MENTOR-ONLY FEATURES IN GITCORD
================================

These features require the "issue_assignees" role(s) configured in config.yaml under:
  assignments:
    issue_assignees:
      - "Mentor"  # (or whatever role name you configure)

1. /assign-issue COMMAND
   - Description: Assign a GitHub issue to a Discord user
   - Requires: issue_assignees role
   - What it does:
     * Parses GitHub issue URL
     * Shows confirmation embed with issue details
     * Allows mentor to confirm or cancel assignment
     * Assigns issue on GitHub after confirmation
     * Sends DM notification to assignee
   - Error if non-mentor tries: "Permission denied. Only mentors with roles [role names] can assign issues."

2. /issue-requests COMMAND
   - Description: List and review pending issue assignment requests from contributors
   - Requires: issue_assignees role
   - What it does:
     * Shows list of repos with pending requests
     * Mentor selects a repo to see requests
     * Shows detailed embed for each request with:
       - Contributor info and activity
       - Issue details
       - Eligibility status
     * Mentor can approve, replace existing assignee, or reject
   - Error if non-mentor tries: "Only mentors ([role names]) can review requests."

3. /sync COMMAND
   - Description: Manually trigger GitHub event sync and notifications
   - Requires: issue_assignees role
   - What it does:
     * Runs the full orchestrator.run_once() process
     * Ingests GitHub events
     * Updates scores
     * Plans role changes
     * Sends notifications
     * Writes snapshots (if enabled)
   - Error if non-mentor tries: "Permission denied. Only mentors with roles [role names] can sync."

4. ISSUE REQUEST REVIEW ACTIONS (within /issue-requests flow)
   - These buttons appear when mentor views a pending request:
   
   a) "Approve & Assign" button
      * Assigns issue to the requesting contributor
      * Updates request status to "approved"
      * Sends DM to contributor
      * Logs audit event
   
   b) "Replace Existing Assignee" button
      * Unassigns current assignee
      * Assigns issue to requesting contributor
      * Updates request status to "approved"
      * Sends DM to contributor
      * Logs audit event
   
   c) "Reject Request" button
      * Updates request status to "rejected"
      * Sends DM to contributor explaining rejection
      * Logs audit event
   
   d) "Cancel" button
      * No action taken, just closes the view

FEATURES AVAILABLE TO ALL USERS (NOT MENTOR-ONLY)
==================================================

- /link - Link Discord to GitHub account
- /verify-link - Verify GitHub link
- /verify - Check verification status
- /status - Show user status and roles
- /summary - Show contribution summary
- /identity - Show identity linking status
- /unlink - Unlink Discord from GitHub
- /pr-info - Show PR context preview (anyone can use)
- /request-issue - Request to be assigned to an issue (contributors)

CONFIGURATION
=============

Mentor roles are configured in config.yaml:

assignments:
  issue_assignees:
    - "Mentor"        # Discord role name(s) that can use mentor features
  review_roles:
    - "Maintainer"    # Roles eligible for PR reviews (different from issue assignment)
  issue_request_eligible_roles: []  # Optional: roles required for contributors to request issues
